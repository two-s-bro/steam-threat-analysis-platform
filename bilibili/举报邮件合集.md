Hello,

I am reporting an active Steam credential-stealing malware
that hijacks the Steam client's UI to redirect users to a
phishing website. This malware was found on my own machine
on 2026-06-16, and I have since conducted a full reverse

═══════════════════════════════════════
ATTACK SUMMARY
═══════════════════════════════════════

The malware targets the Steam client's JavaScript bundle at:
  steamui/chunk~2dcc5aaf7.js

It injects approximately 110 bytes into the ResolveURL()
function to hijack three URL routing keys:

  HelpAppPage     → nexustechsolution.top/steamhelper
  HelpFrontPage   → nexustechsolution.top/steamhelper
  SupportMessages → nexustechsolution.top/steamhelper.html

The injection uses JavaScript's ?? (nullish coalescing)
operator for silent fallback — all other Steam features
work normally, making detection extremely difficult.

The hijacked URLs include the victim's SteamID64 and avatar
hash as tracking parameters.

The malware also modifies steam.cfg to disable auto-update:
  BootStrapperInhibitAll=enable
  BootStrapperForceSelfUpdate=disable

A watchdog thread checks patch integrity every 42 seconds
and re-applies the injection if Steam repairs the file.

A CEF (Chromium Embedded Framework) injection DLL loads a
phishing toast notification:
  "Steam 客服 — 您客服案件的新回复"
  Case Number: HT6YWQBY4XMF55

The toast uses 1:1 CSS extracted from the official Steam
client's CSS, making it visually indistinguishable from a
legitimate Steam notification.

═══════════════════════════════════════
RECOMMENDED ACTIONS
═══════════════════════════════════════

1. Consider integrity verification for steamui/*.js files
   at client startup (checksum comparison).

2. Consider preventing steam.cfg from disabling auto-update
   via BootStrapperInhibitAll when set by non-installer
   processes.

3. Consider adding nexustechsolution.top and associated
   phishing patterns to Steam's built-in URL filter.

4. The malware reads Steam VDF configuration files
   (loginusers.vdf, config/config.vdf) to extract user
