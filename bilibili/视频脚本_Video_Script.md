## 🔬 第二章 (1:00 - 2:30) — 攻击链路全景

**画面**:
- 打开 `docs/01_攻击链路全景图.md`，或者用 PPT 画 4 个方块
- 每讲到一个 Phase，放大对应的方块
- Phase 0: "免费资源/修改器/外挂" 下载诱饵
- Phase 1: SteelDungeon.exe → 自解压 → 写注册表 → 下载组件
- Phase 2: NexusTechNotify.exe → 强杀 Steam → 篡改 JS → 守护线程
- Phase 3: 钓鱼弹窗 → 点击 → 密码被窃

**口播**:
> 这个病毒的攻击链分为四个阶段。
>
> **Phase 0 — 欺骗传播**。你可能在某个网盘、论坛看到"免费资源"、"破解版外挂"，下载了一个看似正常的 exe。
>
> **Phase 1 — 下载器落地**。这个 exe 叫 SteelDungeon，是用 Python 3.14 打包的。它一运行就会做三件事：把自己解压到 AppData 目录、往注册表写自启动键、然后从 C2 服务器下载后续组件。
>
> **Phase 2 — Steam UI 劫持**。第二阶段的主角叫 NexusTechNotify。它会强杀你的 Steam 进程，然后篡改 Steam 客户端里的一个核心 JavaScript 文件，注入了一小段恶意代码。注入完成之后，它还会写入一个 steam.cfg 文件，禁止 Steam 自动更新——这样被篡改的文件永远不会被官方修复。
>
> **Phase 3 — 钓鱼窃密**。接下来它会往 Steam 的浏览器引擎里注入一个钓鱼弹窗，看起来完全就是 Steam 官方的客服通知。你一点，跳转到一个钓鱼网站，输入密码，账号就被拿走了。

**剪辑要点**:
- 每个 Phase 配一张简图或动画箭头
- 可以用鼠标在 VS Code 里打开文件夹目录展示层级关系

## 🧬 第三章 (2:30 - 4:30) — 110 字节的注入代码有多精巧

**画面**:
- VS Code 打开 `D:\Users\周\Desktop\诈骗项目600\chunk~2dcc5aaf7.js.PATCHED`
- Ctrl+F 搜索 `nexustechsolution`
- 展示那段注入代码（约 110 字节）
- 用鼠标高亮 `_h=` 和 `_s=` 两行
- 高亮 `??` 运算符

**口播**:
> 最让我震惊的是这段注入代码——它在 Steam 的 ResolveURL 函数里，只加了大约 110 个字节。怎么做到的呢？
>
> Steam 原版代码里有一个 `let i = r.url;` 的变量声明。病毒在它后面插入了一行，定义了两个新的 URL 变量：`_h` 指向钓鱼网站的主页面，`_s` 指向备用页面。这两个 URL 里还硬编码了受害者的 SteamID64 和头像哈希——也就是说，每个受害者看到的是专属的钓鱼链接。
>
> 然后它用了一个非常精巧的 JavaScript 技巧——`??` 运算符。它构造了一个对象，把 Steam 的 HelpAppPage、HelpFrontPage、SupportMessages 这三个键，全部指向钓鱼 URL。如果当前请求的键不在这个对象里——比如你点的是商店、库存、好友——`??` 就会走原来的 Steam 逻辑，完全不受影响。
>
> 这是什么概念？你在 Steam 客户端里打开商店、看库存、聊天——一切正常。但只要你点击任何一个"帮助"、"客服"、"案件回复"，你就被送到了钓鱼网站。14 兆的 JS 文件里只改了 110 个字节，你不 diff 对比根本发现不了。

**剪辑要点**:
- 代码区放大，用红色/黄色高亮注入部分
- 给 `??` 运算符一个特写
- 可以旁边画一个小表格: HelpAppPage → nexustechsolution.top

**口播**:
> 代码注入讲完了，我们来看看钓鱼弹窗本身。
>
> 这个 HTML 文件就是我电脑上提取出来的。你看它的样子——Steam 原版的齿轮图标、Steam 的 Motiva Sans 字体、渐变背景、悬停发光效果。底部的 CSS 注释写得很清楚：169 行样式，全部从 Steam 官方客户端的 CSS 文件里提取出来的，1:1 还原。
>
> 弹窗内容是"Steam 客服 — 您客服案件的新回复"，案件编号是一个伪造的 HT6YWQBY4XMF55。最巧妙的是，右下角的时间是 JavaScript 动态生成的——你什么时候看到它，它就显示当前时间，看起来像刚刚收到的通知。
>
> 你点一下这个弹窗，Steam 客户端就调用 ResolveURL，把你送到钓鱼网站。钓鱼页面用你的 SteamID 和头像生成个性化界面，你输入密码的那一秒，你的账号就没了。

- 展示设备授权管理

**口播**:
> 说完了病毒，说点重要的——怎么保护自己。
>
> 第一，开启 Steam Guard 手机令牌。这是最有效的防线。即使密码泄露，没有手机验证码，对方一样登不了。
>
> 第二，Steam 设置里找到"管理 Steam 令牌"，注销所有你不认识的设备。
>
> 第三，永远不要下载来路不明的"破解版"、"外挂"、"修改器"。我这个病毒大概率就是通过某个修改器传播的。
>
> 第四，如果你怀疑自己中过招，打开注册表编辑器，导航到 CurrentVersion\Run，检查有没有叫 SteamHelper、SteamService、WindowsUpdateService 的可疑自启动项。
>
> 完整的排查方法、检测 YARA 规则、清理指南，我放在 GitHub 仓库里了，链接在评论区。

**剪辑要点**:
