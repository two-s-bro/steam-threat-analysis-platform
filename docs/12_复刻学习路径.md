## 分模块学习方案

### 模块 1: 钓鱼 UI (难度: ⭐ 低)

**学习目标**: 如何 1:1 还原 Steam Toast UI

**可研究文件**:
- `toast_window.html` — 完整 HTML 结构
- `toast-authentic.css` — 169 行 1:1 还原 CSS
- `desktop_toast_default.wav` — Steam 原生通知音效

**练习方向**:
1. 理解 Steam Toast 的 CSS Module 混淆类名体系
2. CSS 渐变背景、悬停效果复制
3. Motiva Sans 字体加载机制

---

### 模块 2: VDF 解析 (难度: ⭐⭐ 低-中)

**学习目标**: 如何读取 Steam 本地配置文件

**可研究文件**:
- `lib/vdf/` — Python VDF 解析库

**VDF 格式** (Valve Data Format):
```
"key"
{
    "subkey"    "value"
}
```

**Steam 关键配置文件**:
- `Steam/config/loginusers.vdf` — 登录用户信息
- `Steam/config/config.vdf` — 客户端配置

**练习方向**:
1. 编写 Python VDF 解析器
2. 读取自己的 Steam 配置（学习用）

---

### 模块 3: JS 补丁注入 (难度: ⭐⭐⭐ 中)

**学习目标**: 如何篡改 Webpack chunk 并注入代码

**可研究文件**:
- `chunk~2dcc5aaf7.js.PATCHED` — 已篡改的 14MB JS
- 对比 Steam 原版 `chunk~2dcc5aaf7.js`

**练习方向**:
1. `diff` 原版 vs 篡改版，定位所有差异字节
2. 理解 Steam 的 `ResolveURL()` 函数架构
3. 编写自动化 patch 工具

---

### 模块 4: C2 协议 (难度: ⭐⭐⭐ 中)

**学习目标**: 恶意软件如何与 C2 通信

**可研究文件**:
- `_downloadlog/76561198892973993.txt` — C2 通信日志
- `SteamPatchToast_patch.lo` — 完整操作日志

**练习方向**:
1. 还原 C2 通信协议格式
2. 心跳/注册/上报的三阶段模型
3. Cloudflare 隐藏真实 IP 的手法

---

### 模块 5: PyInstaller 打包 (难度: ⭐⭐⭐ 中)

**学习目标**: 如何将 Python 程序打包为独立 exe

**可研究文件**:
- `SteelDungeon.exe` — PyInstaller 打包产物
- `python314.dll` — Python 运行时
- `library.zip` — 标准库压缩包
- `lib/` — 依赖库目录

**练习方向**:
1. 使用 `pyinstxtractor` 解包 exe
2. 提取 `.pyc` 字节码
3. 使用 `uncompyle6` / `pycdc` 反编译

---

### 模块 6: CEF 注入 DLL (难度: ⭐⭐⭐⭐ 高)

**学习目标**: 如何向 Chromium Embedded Framework 注入内容

**可研究文件**:
- `locale_patch.dll` — CEF 注入 DLL

**需要的工具**:
- IDA Pro / Ghidra — 静态反汇编
- x64dbg — 动态调试
- Process Monitor — 观察注入行为

**练习方向**:
1. 识别 DLL 入口点和导出函数
2. 分析注入方法 (SetWindowsHookEx / CreateRemoteThread / DLL 劫持)
3. 分析 CEF 渲染管线劫持点

---

### 模块 7: 注册表持久化 (难度: ⭐⭐ 低-中)

**学习目标**: Windows 自启动机制的攻防

**已分析的内容**:
- 三层 Run 键冗余设计
- 伪装命名策略 (SteamHelper/SteamService/WindowsUpdateService)
- 随机 hash 目录躲避检测

**练习方向**:
1. 研究所有 Windows 持久化位置
2. 编写检测/清理工具

---

### 模块 8: 加密载荷分析 (难度: ⭐⭐⭐⭐ 高)

**学习目标**: 分析加密恶意载荷

**可研究文件**:
- `payload.bin` (38 KB)
- `ungeond.rar` (15.8 MB)

**分析路径**:
1. 从 SteelDungeon.exe 中提取解密密钥/算法
2. 推测使用 AES/RSA (根据 Crypto 库依赖)
3. 解密后分析载荷内容

---

## 推荐学习顺序

```
第1周: 模块 1 (钓鱼UI) + 模块 2 (VDF 解析)
第2周: 模块 7 (注册表) + 模块 4 (C2 协议)
第3周: 模块 3 (JS 注入) + 模块 5 (PyInstaller 解包)
第4周: 模块 6 (DLL 逆向) + 模块 8 (载荷解密)
```
