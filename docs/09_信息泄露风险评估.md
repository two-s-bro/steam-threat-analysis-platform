## 潜在风险场景

### 场景 1: 已点击钓鱼弹窗
如果用户在 Toast 弹出后点击并输入了 Steam 密码:
- 攻击者掌握完整凭证
- 可登录 Steam 账户
- 可发起交易/转移库存

### 场景 2: 仅感染未点击弹窗
- 攻击者仅获取 SteamID64 和哈希
- 无法直接登录
- 但已建立攻击基础设施，可能后续推送更多钓鱼

### 场景 3: 二次传播
- C2 可能下发指令让已感染的客户端传播给 Steam 好友
- 通过 Steam 聊天发送钓鱼链接

## 建议措施

1. **立即修改 Steam 密码** (已完成重装)
2. **开启 Steam Guard 手机令牌** — 最有效的保护
3. **注销所有设备授权** — Steam 设置 → 账户明细 → 管理 Steam 令牌
4. **检查库存/交易记录** — 查看是否有异常交易
5. **检查 API 密钥** — https://steamcommunity.com/dev/apikey
6. **检查已授权登录的第三方网站** — https://steamcommunity.com/id/xxx/edit/settings
7. **举报 C2 域名** — 向 Cloudflare / 域名注册商举报 nexustechsolution.top
